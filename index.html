Basic Understanding of how the internet works

The Web refers to the 'World Wide Web', which is a collection of HTML (Hyper Text Markup Language) documents. Hyperlinks between pages create The Web. The major pieces of The Web are: people, computers, browsers, and the Internet. HTTP (the main protocol of the web) connects servers to the internet. Servers are optimised for hosting files. The internet uses HTTP to communicate with you, your browser, and servers. The HTML is the text content (what you see), markup (what content looks like), References to other documents (videos, images) and links to other pages. HTML code is a 'structured' coding language. This means that it is made up of elements with tags and attirbutes in a 'tree' like structure, called the "DOM" (Document Object Model). This structure allows the computer to read and understand the code, and produce a consistent output.

CSS

CSS stands for 'Cascading Style Sheet'. It is used to apply formatting to your HTML page. The most specific rules are applied, and the code will keep going down and down into the rules, to the one that best describes the element, and that rule gets applied. You can write a selector, or select an HTML tag to apply styles to. For example:

.description {
    color:red;
}

You can change anything to do with the formatting, and standardise it through CSS. It is also possible to have multiple CSS files, so you can start with a library such as Boostrap, and then use your own for customisations.

DYR (Don't Repeat Yourself!)

DYR stands for 'Don't Repeat Yourself. Be avoiding repetition in code, programmers save themselves time, and minimise errors. This is because if the same code is stored in multiple places, it means each update needs to be made multiple times, which either wastes time, or leads to inconsistent code. By storing code in one place, and referencing the same code multiple times, programmers can create more efficient and maintainable code. CSS is just one way that web developers can stay true to the DYR rule, by applying classes to similar div's, it allows formatting to stay consistent. Another example is JavaScript functions, or applying the same .JS file to multiple pages.
